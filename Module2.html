Mobile App and Web App communicate with the backend through RESTful APIs.
The Backend handles the core logic, including device management, automation rules, and user preferences.
Message Queue facilitates real-time messaging between the backend and IoT devices using MQTT.

Cloud Infrastructure handles scalability, ensuring smooth performance even with multiple users and devices.

3.4.8 Internet Router / Wi-Fi Devices / Remote Management
Responsibilities:
Provides network connectivity for devices with the SmartHome Plus ecosystem.
Facilitate internet access for remote control and monitoring via the Hub.
Allow users to control and monitor smart home devices from remote locations via the web portal or mobile app.
Interfaces:
Wireless Communication: Wi-Fi protocol for devices to connect to the hub and broader network.
Port Forwarding: Opens port on the router for secure connections outside the network allowing remote access from outside the local network.
Internal Structure:
Router core will handle Ip assignments, bridge local network with outside of network connections and ensure stable device connections.
On Device chips will handle the connection between the devices and the router.
3.5 Deployment and Operations Strategies

Containerization: Use Docker to containerize services, ensuring portability and simplifying deployment across different environments.

3.5.3 Automation:
Automate infrastructure provisioning using Terraform or AWS CloudFormation to ensure quick and consistent deployment of infrastructure.
